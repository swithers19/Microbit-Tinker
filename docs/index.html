Embedded System?

Embedded systems are a complete computer system that perform certain things within a bigger system (mechanical or electronic). Essentially, its a bunch of code run on a special type of computer which can interact with the world and do some pretty cool stuff.

Spot (Boston Dynamics)

The most exciting embedded/Mechatronic system are robots! Here is a personal favourite of Boston Dynamic's Spot.

Medical Devices

Medical devices such as pacemakers, ventilators and anything with "smarts" processes vitals and determines what must be delivered to a patient. If you are interested in seeing what this code looks like, Medtronic have released the source code and drawings which can be found here.

Planes, Trains and Automobiles

Pretty much everything from your car to your washing machine has an embedded system. A challenge could be to find a home appliance that doesn't feature an embedded system.

Welcome to Microbits

Microbits are the fun little device which can be seen to the right which are an embedded system in one tiny little form factor! It packs a processor, some buttons and sensors (inputs) and a pretty neat LED grid (output). It also has digital and analog IO, which means we can program it to control things based on how we provide or read electricity from the labelled pads.

What Can Be Done

The Microbit is stacked with functionality, there are countless options before you have even connected something to it. It can write, display images, generate music and speech, take motion commands (think Wii remotes), help you navigate old school style and even communicate between microbits using Radio! Nuts, I know.

Below is a simple outline of how we can use some of this functionality with links to the underlying documentation. Get hands on and see how these things interact!

Display

We are already familiar with the display, but it is a class written to make working with those 25 LEDs a little simpler. As we have previously explored we can display.scroll() which will print some text out.

We have also seen display.show() used to display a predefined image. It turns out we can also create and show our own. In the code example you can see img_boat defined with a bunch of numbers. They represent each row of the microbit LEDs, set with a value from 0-9 representing brightness. We then pass this variable to display.show(). Have a crack making your own!

Buttons

Typically we will want to click a button to trigger some sort of event. We can do this using an if statement which checks the state of the button. button_a.is_pressed() checks if the button is currently pressed while there also functions for checking if it has been pressed and how many times.

All functionality is the same for button b in which you swap out the button reference.

Accelerometer

The microbit comes packed with an accelerometer which allows us to detect the orientation of the Microbit. The orientation is represented by cartesian coordinates represented by x, y and z (think back to year 9 maths). We can read each axis value using accelerometer.get_x(), subbing out x for the respective axes.

Microbit makes a super simple interface available to utilise the accelerometer. We can extract the current 'gesture' which is a predefined orientation of the microbit. The gesture can be extracted with accelerometer.current_gesture() and stored in a variable. Check out the example code to see how we could use a gesture to perform a certain action.

Digital IO

Digital Input/Output(IO) provides a simple interface to connect external digital devices such as sensors or peripherals such as LEDs. There are three digital IO pins labelled 0, 1 and 2 which can be programatically read and set. We can also check whether a pin is physically touched using pinX.is_touched().

To output a voltage to a pin (turn on), pinX.write_digital(1) will do the trick, while we can read a pin with pinX.read_digital().

Radio

Radio allows microbits to talk to all their friends! It uses a super simple interface to send and recieve messages between microbits. You will need to import radio and run radio.on() before your loop but then your good to go. To send a message, it is as simple as radio.send(message) and recieve it with radio.receive().

We can specify the channel to communicate on walkie-talkie style so if you want to create your own secret messaging system go for it!

Music

We can generate music from the microbit which requires a simple connection to the stereo connector. We simply connect the ground to the of the cable and the pin connector as seen in the speaker connector diagram if we were to use pin 0.

To actually play music once connected is very simple with music.play()You can use pre-built melodies or build your own. There is a lot of functionality I haven't touched on, check out the documentation if your interested in digging in!
