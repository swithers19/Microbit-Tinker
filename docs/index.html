Messing with Microbit - Intro to Embedded Systems

Code is used in more than just websites, we will look at some of the other ways you can use software to interact with the real world.

  • Embedded System?

Embedded System? Sounds Scary/Boring.

Embedded systems are a complete computer system that perform certain things within a bigger system (mechanical or electronic). Essentially, its a bunch of code run on a special type of computer which can interact with the world and do some pretty cool stuff.

Spot (Boston Dynamics)

The most exciting embedded/Mechatronic system are robots! Here is a personal favourite of Boston Dynamic's Spot.

Ventilators

Medical devices such as pacemakers, ventilators and anything with "smarts" processes vitals and determines what must be delivered to a patient. Due to that rather signifigant 2020 event, Medtronic have released the source code and drawings which can be found here.

Planes, Trains and Automobiles

Pretty much everything from your car to your washing machine has an embedded system. A challenge could be to find a home appliance that doesn't feature an embedded system.

Welcome to Microbits

Microbits are the fun little device which can be seen to the right which are an embedded system in one tiny little form factor! It packs a processor, some buttons and sensors (inputs) and a pretty neat LED grid (output). It also has digital and analog IO, which means we can program it to control things based on how we provide or read electricity from the labelled pads.

To the editor!

The microbit can be program in Python which you are familiar with. The link above takes you to an editor which has a bit going on, so below is a little explanation of what is going on. Don't hesitate to start try uploading code and changing things, nothing will blow up... I hope.

Connect

First thing we need to do is connect to the Microbit. This will allow us to send commands either through flashing our code or through the serial.

Load/Save Code

Given flashing code converts our code into different form, it is ideal to store or code somewhere. This allows you to upload or download your code.

Flash

We can't simply "run" our code, we need to flash it onto the microbit. This compiles our code into a form the Microbit can handle and run for as long as we want it to.

Open Serial

So you are curious about what running the command speech.sing() will do, you can test it out through the serial! This is like the python command line in which you can run one off commands, but on the microbit.
